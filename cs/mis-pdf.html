Content design is more than just copy

A tiny label change with big implications for behaviour, compliance and system design.

Content design UX writing Mortgages Financial services

Context: In the MAE broker journey, the MIS had to be shared with the client at a specific point. A policy change meant it could no longer be pre-generated and downloaded, as originally intended – it now had to be created on demand. The screen was already in build, so stakeholders wanted to make the smallest possible change by simply updating the button label. But this new requirement also changed the interaction, so I had to make the case for adding messaging to keep users informed of what was happening.

Role: I was the sole content designer, responsible for the wider mortgage journey of which this pattern was one small part.

Outcome: The build was updated so the interface matched the underlying logic, and users couldn’t progress without viewing the MIS – removing the policy risk while keeping the experience clear and simple.

1. Original design

The MIS was automatically created in the background and offered as a PDF download. This breached Barclays’ policy – the document should only be created on demand, for viewing on-screen or printing.

Why it matters: the label ‘Download’ no longer matched what the system should do, so it had to change.

2. Policy change

Stakeholders suggested switching ‘Download’ to ‘View’. But the MIS was no longer pre-created – ‘View’ now triggered an API call to generate the document. Viewing was also now mandatory, not optional.

Why it matters: a label change alone didn’t reflect the new behaviour or prevent users moving on without viewing the MIS.

3. Showing system status

Because the preview opened in a new browser tab (often in the background), users needed clear feedback that the MIS had been created successfully.

Why it matters: without confirmation, users might think nothing happened – breaking visibility of system status.

4. Managing failure gracefully

If the API call failed, users had to know. Otherwise they’d click ‘View’, see nothing, and waste time hunting through tabs.

Why it matters: good UX means telling users when something’s gone wrong and what to do next.

5. Enforcing mandatory creation

Sharing the MIS was compulsory. Relying on a label or message wasn’t enough, so the journey blocked progress until the MIS had been opened.

Why it matters: this ensured compliance and a clear user flow.

Outcomes and impact

Tracing the knock-on effects of a ‘simple’ change is core to content design. Without that, user experience – and in this case, compliance – can easily suffer.

This small example shows how content design connects language, behaviour and policy, making sure the system does what the words say – and the words describe what the system actually does.
